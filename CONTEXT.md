# Autolab — Domain Context

This file defines the domain language for the Autolab project. Use these terms consistently in code, docs, ADRs, and conversations.

## Project

**Autolab** — a custom, learning-first homelab scaffold: versioned docs and bash automation for a Proxmox hypervisor, leading into infrastructure-as-code and CI/CD.

## Core Concepts

### Bootstrap

The phase where a bare-metal host gets online manually: ISO install, local console, copy scripts via `scp`, run the network wizard. Bootstrap happens **before** GitOps. The host must have a working uplink before any automation can reach it.

- **Bootstrap scripts** — bash scripts in `docs/proxmox/scripts/` that run directly on the Proxmox host
- **Network env** — the file `/etc/default/proxmox-network.env` containing per-host network config (SSIDs, IPs, interface names). Not committed to git; each host has its own.
- **Network env schema** — `docs/proxmox/config/network-env-schema.yaml`, the single source of truth for network env fields. Adapters are generated from it, and bootstrap scripts iterate its generated key list to apply defaults and render config.

### GitOps

The practice of declarative config in git with a controller that applies it. In Autolab, GitOps starts **after** bootstrap — once the host can reach GitHub Actions via Tailscale.

- **Stack** — a Terramate-managed OpenTofu environment directory under `infra/stacks/`. Each stack defines one environment (e.g., `lab`).
- **Machine** — a VM or LXC resource defined in a stack's `machines` variable. Each machine has a `type` (`vm` or `lxc`) and type-specific config.

### Connection

The Proxmox API connection concept: endpoint URL, API token, TLS mode, node name. Defined once in `infra/connection-schema.yaml` and adapted to OpenTofu, Packer, and CI.

- **Connection adapter** — a generated file that satisfies the connection interface for a specific consumer (OpenTofu module, Packer variables, CI environment variables). Generated by `scripts/generate-connection-adapters.sh`.
- **Connection CI vocabulary** — schema fields use `ci_env` for the target environment variable name and `ci_source` to indicate whether the value comes from a GitHub Environment secret or repository variable.
- **Connection schema** — `infra/connection-schema.yaml`, the single source of truth for Proxmox connection fields. All adapters are generated from it.

### Infrastructure Modules

- **proxmox-connection** — validates and normalises Proxmox connection config. Does NOT create resources; provides plan-time validation and derived values.
- **proxmox-compute** — creates VMs (`type = "vm"`) or LXCs (`type = "lxc"`). Shared concepts (network, CPU, memory, disk, SSH, IP, tags) are defined once; type-specific fields branch internally.
- **cloud-init** — composes cloud-init user-data from base config and optional modules (Tailscale, monitoring, etc.).

### CI Pipeline

- **Preflight** — validates the stack directory exists and computes paths.
- **Pipeline setup** — connects Tailscale, configures Proxmox/R2 env vars, installs OpenTofu + Terramate, inits the stack.
- **Plan** — runs `tofu plan` and uploads the plan artifact.
- **Apply** — runs `tofu apply` with retry logic.

### R2 Backend

Cloudflare R2 (S3-compatible) bucket for OpenTofu state. Setup scripts in `scripts/` create the bucket and API token.

### Packer

Builds VM templates from ISOs on the running Proxmox host. Connection variables are generated from the Connection schema; template-specific Packer variables are hand-maintained, while their GitHub CI vocabulary is generated from `infra/packer/template-schema.yaml`.

- **Packer template schema** — `infra/packer/template-schema.yaml`, the source of truth for template-specific GitHub CI variable names and secret/variable sources.
- **Packer template adapter** — `.github/actions/configure-packer-template/action.yml`, generated by `scripts/generate-packer-template-adapters.sh`; maps GitHub workflow inputs to `PKR_VAR_*`.
- **Packer pipeline setup** — `.github/actions/setup-packer-pipeline/action.yml`; resolves the template, joins Tailscale, emits Connection and template variables, and installs Packer.

## Key Files

| Path | Role |
|------|------|
| `infra/connection-schema.yaml` | Single source of truth for Proxmox connection fields |
| `infra/packer/template-schema.yaml` | Single source of truth for Packer template CI vocabulary |
| `docs/proxmox/config/network-env-schema.yaml` | Single source of truth for network env fields |
| `scripts/generate-connection-adapters.sh` | Generates OpenTofu and Packer adapters from connection schema |
| `scripts/generate-packer-template-adapters.sh` | Generates the Packer template CI adapter from template schema |
| `scripts/generate-network-env-adapters.sh` | Generates env example and bash validation from network schema |
| `infra/_base/*.tm.hcl` | Terramate code generation for providers and backend |
| `infra/stacks/lab/` | First homelab environment |
| `infra/modules/proxmox-connection/` | Connection validation module |
| `infra/modules/proxmox-compute/` | Unified VM/LXC resource module |
| `infra/packer/` | Packer VM template builds |
| `docs/proxmox/scripts/` | Bootstrap-phase bash scripts |
| `docs/proxmox/scripts/lib/` | Shared bash libraries (pure functions) |
| `.github/actions/` | Reusable composite actions for CI |
| `.github/workflows/` | CI workflows (CI, plan, apply, Packer, scripts) |
